   项目名称    花卉志app
        背景介绍:随着近些年手机端app用户的快速增多以及电脑端用户相
   对app用户占比的下降,线上的店相对于实体店的比重越来越大,所以
   花店客户选择了开发一款线上花社区的app,通过打造社区,实现用户
   黏性的增强,为自己的店面打造一个属于自己的活跃社区.
        功能预览:主要是有收藏评论功能,搜索功能,登录功能为手机号
   验证码直接登录,需要提问与回答的功能, 同时可以对回答的情况进行
   点赞,点赞数量多少决定回答的质量,同时可以多人评论,评论人越多
   决定问题的热度,同时每个人可以看到自己提的问题,专家认证的功能,
   经过认证后会有特殊标志,还有积分商城,每个人可以用自己的积分进行
   优惠购花,每个人也可以发表对花的感悟或心得等文章
        技术选型:本来决定通过php的mvc的模式进行项目开发,鉴于phpmvc
    增加了系统结构和实现复杂性,同时视图需要多次调用才能显示足够的
    数据,决定使用react+node.js的方式开发,主要原因有1.速度快,通过虚拟
    dom进行操作以及模块化的容易调用,维护性高
         技术实现: 1.react+weback打包路径的配置
                   2.通过node.js搭建服务器环境       
           技术障碍及解决1.服务器端口的监听与react代理端口以及react启动
        ,端口之间端口号的关系,服务器端口号与react启动项定义端号一直,
        监听代理端口号自定义
                     3.node搭建服务器选择各种功能不同的综合搭配实现服务器的
         搭建,其中设置静态资源以及对于http请求最终决定使用express,,mysql处理
         对wamp数据库的操作,wamp默认端口号3306,通过babel处理对语法的支持
                    4.手机号码的验证使用正则表达式
                    5模糊查询的sql查询语句对格式非常严格
                    6.对于识别客户登录状态通过localstorge储存用户id,通过多表以及
           限制条件实现准确搜索,并按制定顺序排序
                     7.使用muler+antd 上传图片,需要建一个upload文件夹,upload.single必须为file
                     同时antd组件中upload,name必须为file
                     8.react模块的引入和暴露出,
                     9.react的生命周期函数   componentDidMount页面挂载之后
                     10.state只对当前页有效,react为单向数据流,父组件向子组件传递通过props
                     11.后台数据渲染通过antdList表单
